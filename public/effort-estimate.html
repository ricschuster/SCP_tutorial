The three builders

Same target, same quality bar. Only the starting knowledge changes — and with it, the number.

P1 · Baseline Familiar with the stack and SCP
22–30 days ~4–6 weeks full-time 1× baseline

No learning tax. Effort is real engineering: ~21 ideal days across engine, UI, map editing, and the solver integration (~3 days, the single hardest item), plus ~40% real-world overhead for debugging and dead ends.

Tech stack SCP concepts
P2 · +~40% Knows the stack, new to SCP
30–42 days ~6–8.5 weeks full-time ~1.4×

Same engineering ability, but must learn SCP to teaching fidelity — minimum-set, complementarity, irreplaceability, max-coverage — a 1–2 week ramp with real rework risk from getting the model subtly wrong. The ILP formulation is gated on truly understanding the problem.

P3 · 2–5×, open-ended SCP expert, new to the stack
65–120+ days 3–6+ months 2–5×

The largest gap by far. Domain knowledge saves the model-fidelity work, but a typical R/prioritizr practitioner must learn TypeScript, React, Vite, testing, and especially canvas rendering, Web Workers, and wasm solver integration — realistically beyond a first web project without heavy help, so the top end is open. Many would ship a rougher tool (e.g. R/Shiny) instead.

The lever isn't the UI — it's the solver and a faithful model

Both knowledge gaps bite hardest in the same two places. Whoever can't formulate the MILP or doesn't grasp complementarity spends disproportionately there, which is why P2 and P3 diverge so sharply from P1.

Assumptions worth stating

  • Same quality bar throughout — tests, docs, ADRs, accessibility. Drop that and every number roughly halves.
  • Focused days ≠ calendar days. Part-time evenings and weekends stretch these 2–3×.
  • The design already existed as written docs. A true from-scratch solo build also spends time on product design that these numbers only partly fold in.
  • P3's range is open-ended by nature — the wasm/Worker/canvas architecture is the hard ceiling, and some builders wouldn't reach this exact artifact at all.
