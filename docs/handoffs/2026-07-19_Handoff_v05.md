## Process notes (still true)

- Branch protection is strict: after any PR merges, other open PRs go BEHIND;
  update with `gh api -X PUT repos/ricschuster/SCP_tutorial/pulls/<n>/update-branch`.
- CI runs a Prettier `format:check` over the whole repo (`prettier --check .`),
  which includes new non-source files (e.g. HTML in `public/`) and Markdown. Run
  `npx prettier --write <file>` (or `npm run format:check`) before pushing any new
  file, or CI fails fast.
- `gh pr edit` and `gh issue view` fail on this repo (projects-classic / GraphQL
  deprecations); `gh pr create`, `gh pr merge`, `gh issue create/close`,
  `gh release create/view`, and `gh api` all work.
- The deploy workflow (`deploy.yml`) builds the Vite app and publishes `dist` to
  Pages on every push to `main`; `public/` is copied to the site root. `base` is
  `/SCP_tutorial/` only on build.
- Branch before editing; one short-lived branch per change. No em dashes anywhere.
