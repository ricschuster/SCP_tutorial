Shipped (all merged to `main`):

  `irreplaceability()` runs a seeded Monte Carlo over cost-perturbed greedy solves
  and returns per-unit selection frequency; a "Show irreplaceability" toggle adds
  a light-to-warm heat map beside the priority map on both tabs. Design note

- #49 (closes #31): many-feature scalar display. Landscape grew from 3 to 8

  Beaver (riparian), Bobolink (farmland), Common Loon (waterbird), Red Fox

## Current state

What works:

- Explore: set per-species targets for 8 species, paint one of six covers, watch
  habitat and cost derive and the solver re-solve live; inspect any cell; toggle
  irreplaceability; lock in/out. Habitat shows as a combined scalar map + a
  species spotlight; a feature table tracks all species.
- Method: switch objective (min-set / max-coverage), add a compactness penalty,
  add a connectivity penalty (with an optional same-cover boost), weight species,
  read the cost-target curve, and compare greedy against the exact optimum.
- Guided tour: short (6) + optional full (14) narrative walkthrough that drives the
  real UI (layers, spotlight, connectivity) and restores state on close.
- Share/bookmark the full scenario (cover edits, targets, method options including
  connectivity, active tab) via the URL hash.
- 91 tests pass; engine coverage well above the 80% gate. Every PR auto-merged
  after CI.

Architecture (unchanged shape):

- `src/engine/` pure: types, greedy (now with weights, boundary, and connectivity
  options), exact (glpk.js), attainment, compare, feasibility, irreplaceability,
  constants.
- `src/data/`: land-cover (8-species suitability matrix, derivation, explainCover),
  scenario (30x30 landscape, WorkingUnit, combined-habitat helpers, NEIGHBORS),
  connectivity (bounded-radius distance-decay c_ij with same-cover boost), share
  (codec, includes connectivity).
- `src/ui/`: App (tabs, brush, inspector, tour driver with snapshot/restore),
  GridView (canvas), grid-draw (pure helpers), CostTargetCurve, TourPanel, tour
  data, color.

Known limits / simplifications:

- The exact solver now takes ~10s and returns a time-limited (not provably
  optimal) result with 8 species (#53). The guided tour's greedy-vs-exact step
  therefore points at the "Compute exact optimum" button instead of auto-running
  it.
- `vite.config.ts` keeps `testTimeout: 20000`: the full-app UI tests re-solve the
  900-unit, 8-feature landscape on mount and interaction, which is heavy in jsdom
  on CI. This is compute cost, not the old SVG rendering cost.
- Synthetic data; greedy is not optimal (the teaching point); the landscape is not
  a real Canadian ecoregion (a prairie-ish bison sits beside a wetland/loon area).

## Process notes (carried from v02, still true)

- Branch protection is strict: after any PR merges, other open PRs go BEHIND;
  update with `gh api -X PUT repos/ricschuster/SCP_tutorial/pulls/<n>/update-branch`
  (`gh pr update-branch` is not in this gh version).
- Run the dev server as a real background job, never inline `&`.
- Branch before editing; one short-lived branch per change.
- No em dashes anywhere.
- Verify UI-affecting changes in a real browser (Playwright + the bundled
  Chromium is available); jsdom has no 2D canvas context and no layout, so it
  cannot show that pixels render or that pointer-driven interactions work. This
  session used short throwaway Playwright scripts (run from the repo root so module
  resolution works) to verify canvas rendering, drag painting, connectivity, the
  spotlight, and the full tour.
