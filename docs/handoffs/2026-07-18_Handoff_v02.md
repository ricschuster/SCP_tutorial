Second build session. Starting from the completed MVP (see v01), this session
turned the M5 hands-on testing feedback into a "v2" plan and then shipped all of

landscape whose mechanics are visible. Every change went through a PR gated by CI
and squash-merged; the app is live and all planned issues are closed.

## Current state

What works

- Explore: set per-species targets, paint one of six land-cover classes, watch
  habitat and cost derive and the solver re-solve live; inspect any cell for the
  arithmetic; lock in/out with a legend.
- Method: switch objective (min-set / max-coverage under budget), add a
  compactness penalty, weight species, read the cost-vs-target curve, compare
  greedy against the exact optimum, and read the suitability / cost rule tables.
- Guided tour (10 steps) walks land cover -> habitat -> cost -> targets ->
  prioritize -> tradeoff -> constraints, switching tabs and opening the inspector
  as needed.
- Share/bookmark the full scenario (cover edits, targets, method options, active
  tab) via the URL hash.
- 58 tests pass; engine coverage ~99% lines / ~90% branches (gate 80%). Full gate
  green; every PR auto-merged after CI.

Architecture

- `src/engine/` unchanged and pure: types, greedy (with options), exact (now
  glpk.js), attainment, compare, feasibility, constants.
- `src/data/land-cover.ts` (cover classes, suitability, derivation, `explainCover`),
  `src/data/scenario.ts` (30x30 landscape gen, `WorkingUnit`, `applyCover`,
  `toProblemFromUnits`, `NEIGHBORS`), `src/data/share.ts` (v2 codec, includes view).
- `src/ui/`: App (tabs, brush, inspector, tour driver), GridView (fill/border/
  onPaint/onInspect), CostTargetCurve, TourPanel, tour data, color.

Known limits / simplifications

- 30x30 is rendered in SVG (~5.4k rects across the maps), which is render-heavy;
  `vite.config.ts` carries a temporary `testTimeout: 20000` because of it. Canvas
  rendering is issue #31 and should let that timeout drop.
- Feature display is still per-species small multiples (fine for 3 species). The
  many-feature scalar-layer display is part of #31.
- Synthetic data; greedy is not optimal (the teaching point the Method comparison
  makes); the exact solver is bounded to the small grid.

Process notes / gotchas

- Branch protection is strict: after any PR merges, every other open PR goes
  BEHIND and its armed auto-merge will not complete until updated. `gh pr
  update-branch` is not in this gh version; use
  `gh api -X PUT repos/ricschuster/SCP_tutorial/pulls/<n>/update-branch`.
- Run the dev server as a real background job, never inline `&` (it detaches and
  reports "completed" while Vite keeps running orphaned).
- The `react-hooks/set-state-in-effect` rule is enforced: set state in event
  handlers, not effects (the tour's tab switch is done in the step handler).
- Branch BEFORE editing. Twice this session a change was started on the wrong
  branch (an already-merged branch, then `main`); both were recovered
  (cherry-pick / stash onto a fresh branch), but it is avoidable.
- No em dashes anywhere per the working rules; the codec keeps a clean URL for the
  default scenario.
