## Process notes (still true, plus one new)

- **Deploy trigger can silently miss (new this session).** The `push`-to-`main`
  trigger dropped for the #66 and #67 merges: no CI-on-main and no Deploy run were
  created (while `pull_request` CI still fired), so the live site kept serving the
  old bundle. It self-recovered by #68. It was a transient GitHub-side outage, not
  config. Standing practice: after every merge, confirm a deploy run exists for the
  new HEAD (`gh run list --workflow=deploy.yml`); if not, `gh workflow run
  deploy.yml --ref main`, then verify the served bundle contains the change.
- Branch protection is strict: after any PR merges, other open PRs go BEHIND;
  update with `gh api -X PUT repos/ricschuster/SCP_tutorial/pulls/<n>/update-branch`.
- CI runs a Prettier `format:check` over the whole repo; run `npx prettier --write`
  on new/changed files before pushing or CI fails fast.
- `gh pr edit` and `gh issue view` fail on this repo (projects-classic / GraphQL
  deprecations); `gh pr create/merge`, `gh issue create/close`, `gh release`, and
  `gh api` all work. The commit auto-close keyword ("Closes #NN") did not always
  fire on squash merge (worked for #60, not for #58/#59); close manually with
  `gh issue close` if it stays open.
- Branch before editing; one short-lived branch per change. No em dashes anywhere.
